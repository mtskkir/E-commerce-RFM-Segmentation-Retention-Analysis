# E-commerce: RFM-Segmentation & Retention Analysis

**Роль:** Product / Data Analyst  
**Стек:** Python (Pandas, Numpy), Data Engineering (Outer Joins), RFM Analysis, Datetime Manipulation.

## Описание проекта
Бизнес-анализ исторической базы данных крупного интернет-магазина. Исходные данные представляли собой классическую реляционную структуру (множество раздельных таблиц). 
**Цель проекта:** Создать единую витрину данных (Data Mart) и провести RFM-сегментацию пользователей (Recency, Frequency, Monetary), чтобы выявить ядро лояльной аудитории и найти высокодоходных клиентов в зоне риска оттока (Churn).

## Что сделал:
* **ETL и Data Engineering:** Извлек данные из трех разрозненных таблиц (Customers, Orders, Order Items) и объединил их с помощью `pd.merge()` в единый датасет на 113 000+ транзакций.
* **Feature Engineering:** Преобразовал строковые значения времени в формат `datetime`. Вычислил дельту в днях для каждого клиента, сформировав ключевую метрику Recency (Давность покупки).
* **Агрегация и Группировка:** Использовал методы `.groupby()` и `.agg()` для вычисления метрик Frequency (Частота заказов) и Monetary (LTV / Общая сумма покупок) без применения жесткого кодирования (hardcode).
* **Сегментация (Бизнес-логика):** Написал множественные фильтры для поиска аномалий и целевых бизнес-сегментов. 

## Бизнес-результат:
* **Разделение аудитории:** Доказал на цифрах разницу между B2B-клиентами (высокий Monetary, Frequency = 1) и лояльными B2C-покупателями (высокий Frequency).
* **Реактивация выручки:** Нашел **63 "спящих" VIP-клиента** (сумма покупок >2000 у.е., отсутствие активности >300 дней). Сформирован список ID для передачи в отдел маркетинга с целью запуска retention-кампании (звонки, промокоды). Возврат даже 10% этих клиентов принесет компании существенную финансовую выгоду.
